Reference implementation

This page list libraries and application there are implement for uses cases descript in gematik specifications or to create gematik specifications.

Card and Cardreader handling

The following libraries are use to interact with card readers and health cards from desktop applications such as mobile devices like android phones or iphones. We use the following layered architecture:

OpenHealthCard-Events

This library contains overarching events for several layers. That includes general events for info, warn and error. Additionally request events to get asynchronous input from user application.

https://github.com/gematik/ref-OpenHealthCard-Events

HealthCard Control API

HealthCard Control

This library can be used to realize use cases for interacting with a German Health Card (eGk, elektronische Gesundheitskarte) via a mobile device. This is the main API on the highest level of abstraction of physical access providing functionality to gain access to application data on a healthcard including authentication and verification.

https://github.com/gematik/ref-HealthCardControl
https://github.com/gematik/ref-HealthCardControlKit

HealthCard Control Common

This library contains control implementation overarching functionalities like CardDetector to determine automatically the current present card type and generation and inform about card events. The Events send over EventBus are stored in package de.gematik.ti.healthcard.control.common.events and subpackages. Futhermore functionality for pace and verifyPin handling.

https://github.com/gematik/ref-HealthCardControlCommon

HealthCard Access

This library could use to send smartcard commands to a smartcard connected to card reader.

https://github.com/gematik/ref-HealthCardAccess
https://github.com/gematik/ref-HealthCardAccessKit

CardReader Access

The CardReader Access Library add functionality to use different card reader and provider in your application. The CardReaderAccess API provides a registry for higher layer applications to inform about new connected and disconnected card reader.

https://github.com/gematik/ref-CardReaderAccess

CardReader Provider API

The CardReader Provider API define the general communication interface between application and HealthCardReader to abstract the specific implementation for each hardware card reader. The CardReader Provider for each specific hardware are implemented as Java Service Provider and would load dynamically by ServiceLoader implemented in CardReaderAccess.

https://github.com/gematik/ref-CardReaderProvider-Api
https://github.com/gematik/ref-CardReaderProvider-ApiKit
