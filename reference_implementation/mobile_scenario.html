Card and Cardreader handling

The following libraries provide functionality to interact with card readers and health cards. They can be used by desktop applications and mobile devices covering Android and iOS platform. Following layered architecture is applied to those libraries:

OpenHealthCard-Events

This library contains overarching events for several layers. That includes general events for info, warn and error. Additionally events can be requested to get asynchronous input from user application layer.

https://github.com/gematik/ref-OpenHealthCard-Events

HealthCard Control API

HealthCard Control

This library can be used to realize use cases for interacting with a German Health Card (eGk, elektronische Gesundheitskarte) via a mobile device. This is the main API on the highest level of abstraction of physical access providing functionality to gain access to application data on a healthcard including authentication and verification.

https://github.com/gematik/ref-HealthCardControl
https://github.com/gematik/ref-HealthCardControlKit

HealthCard Control Common

This library contains the implementation control functionality like a CardDetector to automatically determine the card type of the card currently present and the information about card events. The events sent over EventBus are stored in package de.gematik.ti.healthcard.control.common.events and subpackages. Futhermore it contains functionality for PACE authentication and PIN verification handling.

https://github.com/gematik/ref-HealthCardControlCommon

HealthCard Access

This library is used to send smartcard commands to a smartcard connected to card reader.

https://github.com/gematik/ref-HealthCardAccess
https://github.com/gematik/ref-HealthCardAccessKit

CardReader Access

The CardReaderAccess library adds functionality to use different card readers and card reader providers in an application. The CardReaderAccess API provides a registry for higher layer applications to inform about newly connected and disconnected card reader.

https://github.com/gematik/ref-CardReaderAccess

CardReader Provider API

The CardReader Provider API defines the general communication interface between application and HealthCardReader in order to abstract the specific implementation from the hardware of card readers. A CardReader Provider needs to be implemented for each specific hardware as Java Service Provider. It is loaded dynamically by ServiceLoader implemented in CardReaderAccess.

https://github.com/gematik/ref-CardReaderProvider-Api
https://github.com/gematik/ref-CardReaderProvider-ApiKit
